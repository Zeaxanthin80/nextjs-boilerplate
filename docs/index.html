📋 Setup Checklist by Role

👑

For Jose (Project Lead):

Follow the Project Lead Guide completely, then choose your database approach:

Cloud Database Approach:
  1. Complete Vercel Workflow Guide setup
  2. Complete Project Lead Guide administrative tasks
  3. Set up cloud database using Cloud Database Setup Guide
  4. Configure team environment variables → Environment Guide
  5. Share development credentials with Norma and Lily securely
Local Database Approach:
  1. Complete Vercel Workflow Guide setup
  2. Complete Project Lead Guide administrative tasks
  3. Set up local PostgreSQL using Local Database Setup Guide
  4. Configure team environment variables → Environment Guide
  5. Create individual database users and share credentials with team
👥

For Norma & Lily (Team Members):

After Jose completes his setup:

  1. Read first: Vercel Workflow Guide (understand the workflow)
  2. Clone repository: git clone <repo-url>
  3. Install dependencies: npm install
  4. Set up environment: Follow Environment Variables Guide
    • Copy .env.example to .env.local
    • Get credentials from Jose (sent via secure email)
    • Fill in your personal .env.local file
  5. Generate Prisma client: npm run db:generate
  6. Test setup: npm run dev
  7. Verify database connection: node test-database.js

🆘 Getting Help

🔍 Problem-Solving Flow:

  1. Identify your issue type:
    • 🌐 Vercel/Deployment issues → Vercel Workflow Guide - Troubleshooting
    • 🗄️ Database connection issues → Check which database guide you're using:
      • Cloud DB: Cloud Database Setup Guide - Troubleshooting
      • Local DB: Local Database Setup Guide - Troubleshooting
    • ⚙️ Environment variable issues → Environment Variables Guide - Troubleshooting
    • 👑 Team coordination issues → Project Lead Guide - Troubleshooting
  2. Check the specific guide for your issue type (each guide has its own troubleshooting section)
  3. Search existing issues in the GitHub repository
  4. Ask in team chat (Discord/Slack) with context
  5. Create GitHub issue for bugs or feature requests
